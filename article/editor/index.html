Мой выбор текстового редактора

Во время обучения у меня был компьютер под операционной системой windows, учитывайте, что говорю только за свой опыт.

Стандартный windows блокнот

Лично я, но думаю и многие, не рекомендую использовать стандартный блокнот для веб разработки, т.к. существует множество текстовых редакторов, специализированно разработанных для программирования в них.

Специализированные текстовые редакторы имеют ряд преимуществ и дополнительных удобств над стандартным блокнотом

  • Подсветка синтаксиса - выделение ключевых слов, для того чтобы быстро ориентироваться в коде
  • Файловый менеджер - в большинстве текстовых редакторов, можно не выходя сразу создать папку, новый документ, работать сразу в нескольких
  • Терминал (командная строка) для запуска или подключения скриптов
  • Подсказки - шаблоны

Sublime Text

ссылка на скачивание Sublime text

как можно сразу заметить этот текстовый редактор сильно отличается внешне от стандартного блокнота. Приятная подсветка, которую можно вырать под себя, спомощью дополнительных плагинов, слева файловый менеджер, где можно быстро переключаться между файлами, а сверху вкладки как у браузера на открытые файлы. Потребуется не мало времени чтобы привыкнуть к работе с текстовым редактором, всеми его возможностями, не говоря уже о многочисленных плагинах.

VS Code

ссылка на скачивание VS Code

На данный момент, это мой любимый текстовый редактор, оформление, темы, некоторые плагины которые можно установить в Sublime text тут уже установлены из коробки, такие популярные как emmet. С его помощью, в VS Code достатоно написать "html:5" чтобы добавить к файл сразу стандартную разметку.

<!DOCTYPE html>
<html lang="ru">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>

</body>
</html>

Так же очень удобной особенностью является Terminal которого нет из коробки в простых текстовых редакторах, тут я не упоминаю IDE для серьезных программистов. Когда уже будет пора изучать бекенд, освоившиться в VS code, вы сможете понять хватает ли вам его возможностей или стоит рассмотреть более сильные IDE для разработчиков. Так же приятной особенностью можно выделить то что он полностью бесплатный. Так же имеет множество плагинов для установки. Подсветку синтаксиса лично я предпочитаю Monokai

Рабочую область для скриншотов я использовал этого сайта на github. Для начинающих разработчиков важно привыкать к системе git, так как во всех компаниях связанных с IT разработкой и командах разработчиков, эта система используется по умолчанию. Ее удобство в системе контроля версий, то есть вы всегда сможете отследить какие изменения были сделаны в проекте, вами или кем-то другим кто работает над проектом.

Notepad++

ссылка на скачивание Notepad++

На скриншоте демонстрация текстового редактора Notepad++. Чуть не забыл упомянуть еще об одной удобной особенности текстовых редакторов, это возможность работать сразу с несколькими документами. Вы оцените это когда понадобится такая возможность. Она присутствует во всех текстовых редакторох которые я указал кроме блокнота.

Atom

ссылка на скачивание Atom

Еще одним хорошим текстовым редактором от команды Github которую ранее упоминал, можно назвать atom, каких-то серьезных преимуществ над предшествующими текстовыми редакторами я не напишу, в силу того, что не так много им пользовался, но он так же может претендовать на ваш выбор, не стесняйтесь попробовать и его в качестве своего текстового редактора чтобы оценить все преимущества и выбрать наиболее удобный для вас.
